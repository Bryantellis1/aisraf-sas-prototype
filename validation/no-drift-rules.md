# No-Drift Rules

These rules apply to AISRAF SAS Prototype v0.1.2 unless a later Build Package contract explicitly authorizes a narrower change.

- No unlisted files or folders.
- No release artifacts before Build Package 15.
- No diagrams before Build Package 13.
- No run outputs before Build Package 11.
- No prompt content before Build Package 04 except `prompts/README.md`. Build Package 04 authorizes the registry, two family READMEs, and the 23 `*.prompt.md` cards under `prompts/rs/` and `prompts/dfd/` only.
- Build Package 04 prompt cards must use only run-profile variables (`{{run_id}}`, `{{input_root}}`, `{{expected_root}}`, `{{output_root}}`, `{{mode}}`, `{{postback_execution_status}}`, `{{output_destination_mode}}`). No prompt may hardcode `run_id`, `sample_id`, paths, source/destination URLs, `output_destination_mode`, or `postback_execution_status`.
- Build Package 04 prompt cards must write only under `{{output_root}}`. No prompt may write to `config/`, `prompts/`, `skills/`, `prototype-agents/`, `.agents/`, `catalogs/`, `blueprints/`, `templates/`, `samples/`, `diagrams/`, `docs/`, `release/`, `tools/`, or `validation/`. No prompt may write to the old reference workspace.
- Build Package 04 prompt cards must not invent run-profile fields outside `config/run-profile.schema.yaml`. Adding a field requires a Build Package 02 schema change in the same change set.
- Build Package 04 prompt cards must not require a Build Package 05+ artifact (skill contract, PRA specification, `.agent.md` adapter, catalog, blueprint, template, sample, run output, diagram, docs/runbook, release artifact) as a hard read. Future-package references must be marked explicitly future-only and the prompt must not stop on their absence.
- Build Package 04 prompt cards must not claim execution of skills, PRAs, `.agent.md` adapters, Jira post-back, Confluence publication, Rovo execution, MCP execution, runtime, cloud, database, Terraform, or ADK services. Cards must not claim diagram generation or release/export.
- Build Package 04 accuracy scoring is owned by `prompts/rs/13-accuracy-score.prompt.md` only and is gated on `scoring_enabled: true`, `expected_baseline_required: true`, and a non-empty populated `{{expected_root}}`. No other prompt may produce a `qualitative_score` field. Scoring outputs may not edit baselines under `{{expected_root}}`.
- Build Package 04 prompt count is fixed at 23 (14 RS + 9 DFD). The full-chain wrapper `prompts/rs/00-run-full-chain.prompt.md` is `status: planned`, writes no output of its own, and may not claim orchestration or full-chain automation. RS output `12-targeted-questions.md` is produced by `prompts/rs/07-missing-fact-question-generate.prompt.md` together with `09-missing-facts.md`; no separate targeted-question prompt may be created.
- Build Package 04 must not create skill contracts, `skills/skill-registry.yaml`, PRA specifications, `prototype-agents/prototype-agent-registry.yaml`, `.agent.md` adapters, catalogs, blueprints, templates beyond authoring-agent templates, samples, expected baselines, run outputs, diagrams, docs/runbooks, release artifacts, schemas outside `config/`, `.github/prompts/` wrappers, or release-export tools.
- Build Package 04 may extend `tools/Test-AisrafPackage.ps1` only for compatibility (split Check 08 to allow `prompts/` content matching the registry; extend Check 11's allowed validation file list). Build Package 04 must not modify `validation/package-03-tools-checklist.md`, which is frozen Package 03 acceptance evidence.
- No skill content before Build Package 05 except `skills/README.md`. Build Package 05 authorizes `skills/skill-registry.yaml`, `skills/rs/README.md`, `skills/dfd/README.md`, the 17 RS skill contracts under `skills/rs/SK-*.md`, and the 9 DFD subskill contracts under `skills/dfd/SK-DFD-0[1-9]-*.md`.
- Build Package 05 skill contracts must use only run-profile variables (`{{run_id}}`, `{{input_root}}`, `{{expected_root}}`, `{{output_root}}`, `{{mode}}`, `{{postback_execution_status}}`, `{{output_destination_mode}}`). No skill may hardcode `run_id`, `sample_id`, paths, source/destination URLs, `output_destination_mode`, or `postback_execution_status`.
- Build Package 05 skill contracts must declare write paths only under `{{output_root}}` (or `{{output_root}}/dfd/` for DFD subskills) when invoked through their mapped Build Package 04 prompt card. No skill may declare a write to `config/`, `prompts/`, `skills/`, `prototype-agents/`, `.agents/`, `catalogs/`, `blueprints/`, `templates/`, `samples/`, `diagrams/`, `docs/`, `release/`, `tools/`, or `validation/`. No skill may declare a write to the old reference workspace.
- Build Package 05 skill contracts must not invent run-profile fields outside `config/run-profile.schema.yaml`. Adding a field requires a Build Package 02 schema change in the same change set.
- Build Package 05 skill contracts must not require a Build Package 06+ artifact (PRA specification, `.agent.md` adapter, catalog, blueprint, template, sample, run output, diagram, docs/runbook, release artifact) as a hard read. Future-package references must be marked explicitly future-only and the skill must not stop on their absence.
- Build Package 05 skill contracts must not claim execution of PRAs, `.agent.md` adapters, Jira post-back, Confluence publication, Rovo execution, MCP execution, runtime, cloud, database, Terraform, or ADK services. Skills must not claim diagram generation or release/export. Only `skills/rs/SK-ACCURACY-SCORE.md` produces an accuracy score and only when the mapped prompt's scoring conditions are met; no other skill may produce a numeric accuracy score.
- Build Package 05 skill contracts must not modify any file under `prompts/` or any file under `{{expected_root}}`. The Package 04 prompt registry is sealed for the duration of Build Package 05; the canonical Package 05 skill names diverge from two Package 04 prompt-registry placeholder names (`SK-DFD-LEGEND-NORMALIZE` → `SK-LEGEND-NORMALIZE`, `SK-SECURITY-STACK-DETECT` → `SK-SECURITY-STACK-ASSESS`). The drift is documented in `skills/skill-registry.yaml` `compatibility_notes`; no edit to the prompt registry is permitted.
- Build Package 05 skill count is fixed at 26 (17 RS + 9 DFD). The full-chain wrapper placeholder `SK-CHAIN-WRAPPER` referenced by `prompts/rs/00-run-full-chain.prompt.md` is recorded only under `planned_future_skills` in the skill registry and is not counted in the 26.
- Build Package 05 must not create skill-execution code, PRA specifications, `prototype-agents/prototype-agent-registry.yaml`, `.agent.md` adapters, catalogs, blueprints, templates beyond authoring-agent templates, samples, expected baselines, run outputs, diagrams, docs/runbooks, release artifacts, schemas outside `config/`, `.github/prompts/` wrappers, or release-export tools.
- Build Package 05 may extend `tools/Test-AisrafPackage.ps1` only for the four authorized compatibility changes: (a) update header/synopsis text; (b) remove `skills/` from the README-only folder rule; (c) add Check 08c-skills-content-limits enforcing the approved skill-folder layout; (d) extend Check 11's allowed validation-file list to include `package-05-skills-checklist.md` and `skill-registry-checklist.md`. Build Package 05 must not modify `validation/package-03-tools-checklist.md`, `validation/package-04-prompts-checklist.md`, or `validation/prompt-registry-checklist.md`.
- No PRA or `.agent.md` adapter content before Build Package 06.
- No catalogs before Build Package 07 except `catalogs/README.md`.
- No blueprints before Build Package 08 except `blueprints/README.md`.
- No templates before Build Package 09 except Build Package 01 authoring-agent templates.
- No samples or expected baselines before Build Package 10.
- No test claims without evidence.
- No Jira, Confluence, or MCP claims without execution evidence.
- No cloud, GCP, ADK, database, Terraform, runtime, or deployed-agent claims.
- No modification to the old reference workspace.
- No copying old stale release artifacts, failed diagrams, run proof, temporary reports, or generated outputs into this clean rebuild.
- No Build Package may invent a run-profile field outside `config/run-profile.schema.yaml`. Adding, removing, or renaming a field requires a Build Package 02 schema change in the same change set, with matching updates to `config/run-profile.template.yaml`, the samples, `config/run-profile.field-catalog.md`, and `config/run-profile.validation-rules.md`.
- No artifact, prompt, skill, PRA, runbook, scoring report, manifest entry, evidence record, diagram, or message may claim Jira post-back, Confluence publication, MCP execution, or connector use unless `postback_execution_status: executed_by_operator` (or the equivalent connector or MCP `executed_by_operator` value) is set AND `output_root/00-run-log.md` records timestamp, destination URL, and operator role for that action.
- Build Package 03 tools must resolve every run-profile field through `config/run-profile.template.yaml` and `config/run-profile.schema.yaml`. A tool may not invent, rename, or silently default any field outside that schema; adding a field requires a Build Package 02 schema change in the same change set.
- Build Package 03 tools must not write outside `runs/<run_id>/`. They must not modify `config/`, `prompts/`, `skills/`, `prototype-agents/`, `.agents/`, `catalogs/`, `blueprints/`, `templates/`, `samples/`, `diagrams/`, `docs/`, `release/`, the old reference workspace, or any path with a drive letter, leading `/`, or `..` segment.
- Build Package 03 tools must not claim execution of prompts, skills, PRAs, `.agent.md` adapters, Jira, Confluence, Rovo, MCP, scoring, diagram generation, runbook generation, or release export. Console output and run-log entries must describe only what the tool did.
- Build Package 03 must not introduce `tools/Export-AisrafRelease.ps1` or any other release-export tool. Release packaging is owned by Build Package 15 and gated by founder approval.
- Smoke-run folders created by Build Package 03 tooling must be removed before human git stage. No `runs/<run_id>/` folder may be committed before Build Package 11.
